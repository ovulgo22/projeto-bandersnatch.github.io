GET /v1/users

Recupera uma lista paginada de usuários.

Parâmetros de Query

  • limit integer Opcional
    O número de resultados a serem retornados por página. Padrão: 20.
  • offset integer Opcional
    O número de resultados a serem pulados para paginação. Padrão: 0.
  • sort string Opcional
    Campo para ordenação. Ex: `name_asc`.

Respostas

200 OK

Uma lista de objetos de usuário.

401 Unauthorized

Chave de API inválida ou ausente.

POST /v1/users

Cria um novo usuário.

Corpo da Requisição (Request Body)

Um objeto JSON contendo as informações do novo usuário.

Respostas

201 Created

O objeto do usuário recém-criado.

400 Bad Request

Dados inválidos no corpo da requisição.

Exemplos de Código


curl -X GET "https://api.seusite.com/v1/users?limit=10" \
     -H "Authorization: Bearer YOUR_API_KEY"


const apiKey = 'YOUR_API_KEY';

fetch('https://api.seusite.com/v1/users?limit=10', {
  headers: {
    'Authorization': `Bearer ${apiKey}`
  }
})
.then(response => response.json())
.then(data => console.log(data));


import requests

api_key = "YOUR_API_KEY"
headers = {
    "Authorization": f"Bearer {api_key}"
}
response = requests.get(
    "https://api.seusite.com/v1/users?limit=10",
    headers=headers
)

print(response.json())


curl -X POST "https://api.seusite.com/v1/users" \
     -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     -d '{
           "name": "Ada Lovelace",
           "email": "ada@example.com"
         }'
